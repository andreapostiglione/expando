# Architecture (contributors)

High-level map of the Expando Python package (`src/expando/`).

## Runtime flow

```
CLI (cli.py)
  └─ daemon start → subprocess → daemon.foreground()
        └─ listener.run_service()
              ├─ ExpansionEngine (engine.py) — trigger matching & render
              ├─ KeyboardService (listener.py) — pynput + file watcher
              ├─ ListenerWatchdog — restart dead listener thread
              └─ menubar.py (macOS) — rumps UI + update checks
```

## Core modules

| Module | Role |
|--------|------|
| `config.py` | Load/compile YAML matches and app settings |
| `engine.py` | Keystroke buffer, trigger matching, injection |
| `listener.py` | OS keyboard hook, hotkeys, config hot-reload |
| `injector.py` | Type or clipboard injection backends |
| `renderer.py` | Variable expansion, forms, shell vars |
| `plugins.py` | Load `plugins/*.py` hooks (allowlist-aware) |
| `daemon.py` | PID file, single-instance lock, start/stop |
| `paths.py` | Config dir layout and defaults |

## Observability (Sprint 34–35)

| Module | Role |
|--------|------|
| `health.py` | `runtime-health.json` metrics + `expando health` |
| `support_bundle.py` | Redacted tar.gz for support |
| `auto_backup.py` | Scheduled + pre-mutation backups |
| `crash_loop.py` | Safe mode policy + doctor sparkline |
| `crash_reporting.py` | Uncaught exception JSON in `crashes/` |
| `doctor_checks.py` | `expando doctor` text/JSON/HTML |
| `log_viewer.py` | `expando logs` plain or `--json` |

`runtime-health.json` is updated by the listener (expansions, reloads, listener liveness)
and `updater.py` (last update check).

## Hub & marketplace

| Module | Role |
|--------|------|
| `hub.py` | Install/remove local packages |
| `hub_marketplace.py` | Remote index, validation, publish-site HTML |
| `registry_catalog.py` | Built-in package registry |

Maintainer sync is validated weekly via `.github/workflows/hub-maintainer-sync.yml`.

## Sync

`sync_assist.py` inspects git/iCloud symlink state and blocks destructive operations when
git is dirty or iCloud conflict markers are present.

## UI

| Module | Role |
|--------|------|
| `snippet_editor*.py` | Native/AppKit or CLI snippet editor |
| `ui_bridge.py` | Bridge between menu bar and editor |
| `onboarding.py` | macOS permission wizard |

## Testing layout

```
tests/           # Unit tests (pytest)
tests/e2e/       # macOS integration (EXPANDO_E2E_FULL=1)
```

E2E runners: see [E2E_RUNNERS.md](E2E_RUNNERS.md).

## Extension points

1. **Plugins** — `before_expand`, `after_expand`, `transform_replacement` hooks
2. **Packages** — YAML under `match/packages/<id>/`
3. **Profiles** — per-app overrides in `config/<profile>.yml`

## Build & release

- `pyproject.toml` — package metadata and entry point `expando`
- `Expando.app` — signed macOS bundle (release workflow)
- `docs/` — GitHub Pages site generated by `hub portal publish-site`

## Data on disk

```
<config_dir>/
  config/default.yml
  match/
  plugins/
  expando.pid / expando.lock / expando.log
  runtime-health.json
  crash-loop.json
  backups/
  crashes/
```

Default templates ship from `default_config/` on first run (`paths.ensure_default_config`).
